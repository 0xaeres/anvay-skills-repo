# nexus-skill

## Use This Skill When
Use for product orientation, grounded development, review, debugging, and deciding when to query the
product knowledge base.

## Product Snapshot
Nexus is a sovereign, MCP-native context engine that indexes a product's code and documentation to
generate curated product skill packs via a bounded expert LLM council [E1, E4]. It is used by
engineering teams to ground AI coding clients in actual code and conventions [E4]. The primary
runtime is Python 3.13+ with a FastAPI backend [E19]. [file: github:0xaeres/nexus-core/README.md:17]

## Product Language
- **Council**: A bounded LangGraph-based expert LLM group that drafts skill proposals [E1, E14].
  [file: github:0xaeres/nexus-core/README.md:17]
- **Skill Pack**: A curated set of Markdown files (e.g., `{product}-skill.md`) that provide grounded
  context to AI clients [E1, E2].
- **Ingest**: The process of syncing sources, chunking content, and indexing vectors [E3].
- **MCP (Model Context Protocol)**: The native interface used to serve skills and raw corpus to AI
  clients [E1, E4].
- **Proposal**: A draft skill generated by the council that requires human approval before becoming
  a permanent skill file [E1, E15].
- **Hybrid Retrieval**: A search pipeline combining dense and BM25-sparse embeddings with RRF and
  reranking [E3, E14].

## Capabilities And Workflows
1. **Continuous Ingest** - Syncs resources from GitHub or local files, computes SHA-256 diffs, and
   indexes chunks into Qdrant [E3, E4]. [file: github:0xaeres/nexus-ui/DESIGN.md:10]
2. **Skill Synthesis** - Runs a bounded council (Planner, Product Mapper, Engineering Mapper,
   Synthesizer) to draft product-scoped skills [E12, E15].
3. **Human-in-the-Loop Curation** - Allows users to review, edit, and approve proposals via a UI
   before they are written to the skill store [E1, E15].
4. **MCP Context Serving** - Serves approved skills and raw corpus to AI clients like Claude,
   Cursor, or Continue [E1, E4].

## System Map
- **nexus-core**: The backend engine containing the API, ingest pipeline, and council logic [E14].
  [file: github:0xaeres/nexus-ui/DESIGN.md:10]
- **nexus-ui**: The frontend where humans drive sources and approve council proposals [E6].
- **Ingest Pipeline**: Handles chunking (tree-sitter for code, headings for docs) and Qdrant
  indexing [E3, E14].
- **Council (LangGraph)**: Orchestrates the Planner, Mappers, and Synthesizer to create skill drafts
  [E12, E14].
- **Retrieval Pipeline**: Implements hybrid search (dense + BM25) and reranking [E14].

## Data Model
- **Product**: The root entity; all access is scoped to a product to prevent cross-tenancy leaks
  [E3, E9]. [file: github:0xaeres/nexus-ui/components/screens/Dashboard.tsx:26]
- **Resource**: A unit of content identified by a canonical URI and SHA-256 hash [E3].
- **Chunk**: A semantic segment of a resource stored in Qdrant with dense and sparse vectors [E3].
- **Proposal**: A SQLite-queued draft of a skill awaiting human approval [E14, E15].
- **Skill**: An approved Markdown file stored in the skill store [E1, E15].

## Interfaces And Contracts
- `METHOD /products`, `/sources`, `/council`, `/skills`, `/setup`: FastAPI routes for managing the
  Nexus lifecycle [E14]. [file: github:0xaeres/nexus-core/ENGINEERING.md:10]
- `MCP Tools`: Interface used to serve approved skills and raw corpus to AI coding clients [E1, E4].
- `approve_proposal()`: Internal contract that writes a proposal to a final `SKILL.md` file [E15].

## Invariants And Constraints
1. **Human-in-the-Loop**: No write action to a skill file may occur without the proposal/approval
   step [E5]. [file: github:0xaeres/nexus-core/AGENTS.md:83]
2. **Tenancy Isolation**: The system must prevent cross-product access; account-scoped lookup APIs
   are required to prevent boundary crossing [E9].
3. **Dependency Control**: No runtime dependencies may be added without discussion to keep the
   dependency set deliberate [E5].

## How To Use The Knowledge Base
Query the product knowledge base when this skill's high-level summaries are insufficient for
implementation or when you need the latest raw code/doc evidence.
- Use `find_skills` to locate curated guidance and existing skill packs.
- Use `query_code_context` for specific symbol lookups, definition tracing, or checking current
  implementation details.
- Use `hybrid_search_corpus` for open-ended searches across the indexed code and documentation.
This skill is sufficient for architectural orientation and workflow understanding; fresh retrieval
is required for specific API changes or debugging current logic.

## Known Traps
- **Dependency Bloat**: Adding runtime dependencies without discussion violates the deliberate
  design of the system [E5]. [file: github:0xaeres/nexus-core/AGENTS.md:83]
- **Bypassing Approval**: Attempting to write skills directly without the proposal/approval flow
  violates a core product invariant [E5].
- **Demo Seeding**: Seeding placeholder products is forbidden; users must onboard via the wizard
  [E5].
